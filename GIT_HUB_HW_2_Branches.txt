
1. На локальном репозитории сделать ветки для:
- Postman:$ git branch Postman
- Jmeter:  $ git branch Jmeter
- CheckLists: $ git branch CheckLists
- Bag Reports: $ git branch BagReports
- SQL:   $ git branch SQL
- Charles: $ git branch Charles
- Mobile testing: $ git branch MobileTesting

2. Запушить все ветки на внешний репозиторий
$ git push -u origin Postman Jmeter CheckLists BugReports SQL Charles MobileTesting

3. В ветке Bag Reports сделать текстовый документ со структурой баг репорта
  3.1 Переключаемся на ветку: $ git checkout BugReports
  3.2 Создаем файл: $ touch bug_report.txt
  3.3 Вносим изменения: $ nano bug_report.txt
      ID - идентификатор бага
      Environment - окружение
      Title - краткое описание
      Description - Подробное описание
      Steps To Rreproduce  - шаги воспроизведения
      Expected Results - ожидаемый результат
      Actual Results - фактический результат
      Severity - важность бага с точки зрения работы ПО
      Priority - срочность устранения с точки зрения бизнесаgit
      Attachments - приложение (скриншот,видео,лог)
      Date - дата обнаружения бага
      Author - ФИО тестировщика

4. Запушить структуру багрепорта на внешний репозиторий

   $ git add bug_report.txt
   $ git commit -m "Add bug_report.txt "
   $ git push

5. Вмержить ветку Bag Reports в Main
   $ git checkout main
   $ git merge BugReports

6. Запушить main на внешний репозиторий.
   $ git push -u origin main

7. В ветке CheckLists набросать структуру чек листа.
   7.1 Переходим в ветку: $ git checkout CheckLists
   7.2 Создаём файл: $ touch CheckList.txt
   7.3 Вносим изменения: $ nano CheckList.txt
      ID - номер проверки
      Title - название проверки
      Test steps - шаги проверки
      Expected Result - ожидаемый результат 
      Status - статус проверки
      Comment - комментарий 
      Date - дата прохождения
      Author - ФИО тестировщика
   
8. Запушить структуру на внешний репозиторий
   $ git add CheckList.txt
   $ git commit -m "Add CheckList.txt "
   $ git push

9. На внешнем репозитории сделать Pull Request ветки CheckLists в main
   На web-интерфейсе перейти в ветку CheckLists 
   Нажать кнопку Compare & pull request - Сreate pull request

10. Синхронизировать Внешнюю и Локальную ветки Main
   $ git checkout main
   $ git pull
